# GitHub Deep-Dive Mode

Use this mode inside `information-retrieval` when repository structure, issues,
PRs, commits, releases, or architecture evolution are central.

Workflow:

1. Define target repo or search scope.
2. Build a repo baseline from README, tree, language mix, and major modules.
3. Inspect key entrypoints and subsystem boundaries.
4. Reconstruct issue/PR/commit timeline with explicit dates when relevant.
5. Separate current mainline behavior from historical experiments.
6. Synthesize reusable patterns, risks, confidence, and cited sources.

Prefer first-party repository evidence over third-party summaries.
