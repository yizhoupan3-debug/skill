# PPTX Method

Use this reference when the deck needs a more systematic making process instead of isolated beautification tricks.

This reference distills several repeat themes associated with 旁门左道PPT materials:

- overall thinking
- structural thinking
- visualization thinking
- a visual-style formula built from background tone, color consistency, typography, and image style
- layout basics built on contrast, alignment, proximity, and repetition
- a slide workflow of information extraction, hierarchy sorting, visualization, then layout

## Three Thinking Modes

### 1. Overall Thinking

Look at the deck as one communication object, not a pile of individual pages.

- define the overall narrative first
- decide the deck's mood and audience before choosing colors
- make sure the cover, body, and closing feel like one family

If the deck does not read as a whole, local page polish will not save it.

### 2. Structural Thinking

Structure the message before styling it.

Useful structures include:

- pyramid / conclusion-first
- question-answer
- timeline
- comparison
- process flow

For course reports, a practical default is:

1. topic and scope
2. method or case setup
3. evidence or comparison
4. conclusion and discussion

### 3. Visualization Thinking

Ask what the audience should see, not only what they should read.

- turn long prose into a relationship, sequence, comparison, or takeaway
- use one dominant visual form per slide
- if a point is not visual yet, keep simplifying until it is

## Visual Style Formula

Treat visual style as a controlled system, not a random template choice.

Build it from:

1. background tone
2. unified palette
3. scenario-appropriate typography
4. image style consistency

For a high-end course report, a reliable formula is:

- dark cover plus warm content pages
- one accent family only
- title font with more presence than body font
- consistent image treatment such as cinematic crop, archival document crop, or restrained diagram panel

## Layout Order

For each slide, work in this order:

1. information extraction
2. hierarchy sorting
3. visualization choice
4. layout composition

This prevents the common failure mode where a slide is styled before anyone knows what matters.

## Page-Type Guidance

### Full-image slide

- first find the visual focus
- then place text where contrast and reading flow remain strong
- use overlays, panels, or cropping only to support the message

### Table

- remove decorative clutter
- unify alignment
- highlight the important row or column
- move raw detail to appendix slides if needed

### Chart

- identify the single key comparison first
- mute the rest
- restyle colors and labels to match the deck
- never leave an untouched default office chart in a polished deck

### Text-heavy slide

- extract the message into 2-4 blocks
- reduce paragraphs into claims, labels, or takeaways
- if the page still looks like a Word document, the visualization step failed
