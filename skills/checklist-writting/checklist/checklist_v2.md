# Checklist v2 Notes

这个版本在 `cl_v1.md` 基础上强化以下规则：

1. **点和点之间默认并行**。
2. **明确串行的，再长都写在同一点里**，不要拆成多个同级点制造假并行。
3. 每个点都优先包含：
   - 当前状态
   - 目标
   - 独占写入范围
   - 禁止越界
   - 串行步骤
   - 退出条件
   - 停手条件
   - 交付物
   - 验收标准
   - 执行结果
4. 写完 checklist markdown 后，**必须单独告诉用户要开几个 agent**，这个数字等于并行点数量，而不是串行步骤数量。

使用时直接以 `cl_v1.md` 为正文模板；本文件是版本说明与强化规则。
