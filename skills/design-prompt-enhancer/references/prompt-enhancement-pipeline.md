# Prompt Enhancement Pipeline

这层的作用是把用户脑子里的模糊设计想法，翻译成后续模型更稳定能执行的输入。

## 目标

从：

- "做个好看点的首页"
- "高级一点"
- "把按钮改顺眼"

变成：

- 页面类型明确
- 平台明确
- 氛围明确
- 组件明确
- 结构明确
- 设计系统约束明确

## 推荐步骤

1. 判断是新页面生成还是定点编辑
2. 补平台和页面类型
3. 把大白话翻译成 UI 术语
4. 引入 `DESIGN.md` 或现有设计系统
5. 写成固定 prompt 骨架
6. 明确禁区，防止风格漂移

## 为什么这层值钱

- 它减少“同一句话每次生成都不一样”的漂移
- 它能把设计系统真的带进后续生成
- 它把 prompt 从一句话变成半结构化规格
