# Type-Specific Check Dimensions

Apply the appropriate dimension checks based on assignment type.

## Paper / Report Assignments

| # | Check dimension | What to verify |
|---|----------------|----------------|
| D1 | Page / word count | Exact count including/excluding references, appendix |
| D2 | Required sections | Each mandated section (intro, method, results, conclusion, etc.) present |
| D3 | Figure / table count | Meet minimum count, each numbered and captioned |
| D4 | Citation count & format | Minimum references met, format matches required style |
| D5 | Content depth per section | Each section addresses the specific questions or analysis asked |
| D6 | Data / results presentation | All required data tables, statistical tests, comparisons included |
| D7 | Originality | Not recycled from other courses, sufficiently distinct from references |

## Code / Programming Assignments

| # | Check dimension | What to verify |
|---|----------------|----------------|
| D1 | Required functions/classes | Each specified function/class implemented with correct signature |
| D2 | Input/output format | Matches spec exactly (stdin/stdout, file I/O, return types) |
| D3 | Test cases | All provided test cases pass, edge cases handled |
| D4 | Code comments & documentation | Sufficient inline comments, docstrings, README |
| D5 | Algorithm constraints | Specified time/space complexity met, no forbidden libraries |
| D6 | Build & run instructions | Code compiles/runs without errors on target environment |
| D7 | Output format | Output matches expected format precisely (whitespace, ordering, etc.) |

## Math / Problem Set Assignments

| # | Check dimension | What to verify |
|---|----------------|----------------|
| D1 | All problems attempted | Every numbered problem/subpart has an answer |
| D2 | Work shown | Derivation steps present, not just final answers |
| D3 | Notation consistency | Symbols match the course conventions |
| D4 | Units & dimensions | Physical units correct and consistent throughout |
| D5 | Proof structure | Clearly stated assumptions, logical steps, QED |

## Presentation / Slides Assignments

| # | Check dimension | What to verify |
|---|----------------|----------------|
| D1 | Slide count | Within required range |
| D2 | Time allocation | Content matches allocated presentation time |
| D3 | Required content | All mandated topics covered |
| D4 | Visual quality | Readable fonts, appropriate graphics, not text-heavy |
| D5 | Speaker notes / script | Included if required |
