# Design Workflow Reference

Use this compact loop:

1. Capture the design target, audience, and existing assets.
2. Decide whether the job is `capture DESIGN.md`, `prompt generation`, or
   `acceptance verdict`.
3. Keep reference tokens concrete: typography, color, spacing, surface, motion,
   interaction states, and anti-patterns.
4. For acceptance, compare actual output against the agreed tokens and return a
   verdict: pass, rework, or fail.
5. Route screenshot-based evidence to `visual-review` when rendering proof is
   needed.
