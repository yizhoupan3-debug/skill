# DESIGN.md Quality Rubric

一份好的 `DESIGN.md` 至少要过下面 6 关：

## 1. 不是空形容词

- 不只是“现代、简洁、高级”
- 要能说清颜色、层级、圆角、阴影、排版、密度

## 2. 有语义，也有精确值

- 颜色要有角色名
- 关键色最好带 hex
- 不能只有设计感觉，没有约束

## 3. 能复用到下一页

- 不是只解释当前页面
- 要能支持后续页面继续生成

## 4. 有明确禁区

- 要写什么不能出现
- 例如多余渐变、假数据卡片、风格漂移、泛化按钮造型

## 5. 能直接进入 prompt

- 应该能抽出一段 `DESIGN SYSTEM` block 给后续 prompt 直接复用

## 6. 有边界感

- 哪些是高置信规则
- 哪些只是从样本推断出来的倾向
